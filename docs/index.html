Описание идеи

  • Проблема: сложно находить персонализированные подборки фильмов и сохранять любимые в одном месте, реклама.
  • Решение: удобный каталог фильмов с поиском, деталями и списком избранного (локальное хранение).
  • Ключевой функционал: просмотр популярных, поиск, страница фильма, избранное, адаптивный UI.
  • Преимущество: лёгкая кастомизация рекомендаций, офлайн-доступ к избранному, нет рекламы

Пользовательские сценарии

  1. Пользователь открывает главную — видит популярные фильмы — использует поиск по заголовку — открывает страницу фильма.
  2. Пользователь на странице фильма — добавляет в избранное — проверяет список избранного в профиле/странице "Избранное".
  3. Пользователь фильтрует результаты по жанру/году — кликает фильм — делится ссылкой на фильм.

API (REST)

Интеграция через TMDB + собственные endpoint'ы при необходимости.

  • GET /movie/popular?page={n} — список популярных
  • GET /search/movie?query={q}&page={n} — поиск
  • GET /movie/{id} — детали фильма
  • Клиент хранит токен TMDB в окружении; любимые хранятся в localStorage.

Вопросы и ответы

Цель проекта?
Создать удобный и адаптивный каталог фильмов с персональными рекомендациями и возможностью сохранять избранное для офлайн-доступа.
Какие проблемы решает проект?
Сложность поиска персонализированных подборок и хранение избранных фильмов в одном удобном месте.
Какие основные функции предусмотрены?
Просмотр популярных фильмов, поиск по заголовку, фильтрация по жанру и году, список избранного и страница с деталями фильма.
